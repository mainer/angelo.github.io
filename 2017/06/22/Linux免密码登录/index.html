Linux免密码登录

发表于 2017-06-22 | 0 comments | 阅读次数

Linux免密码登录踩坑以及详细分析介绍。

SSH 实际上是一个服务，在服务器上，该服务一直运行。并默认在 22 端口号上侦听请求。其登录方式有两种：帐号密码登录、公钥/私钥免密登录。对于帐号密码登录来讲，直接输入帐号密码即可，每次登录都需要验证密码。对于免密码登录，实际上使用的是公钥私钥验证。

SSH 服务在服务器上的 ~/.ssh 目录和客户端机器的 ~/.ssh、/etc/ssh/ssh_config 目录下的文件读取配置、密钥进行验证。

一般情况下，SSH 会读取 session 中的密钥，然后读取 ~/.ssh/id_rsa 进行验证，验证失败后仔不断尝试其它验证方法，最后都尝试密码登录。

在 ~/.ssh/known_hosts 中存储有 host 与 publicKey 的对应关系，ssh 认为每个 publicKey 对应一个 host，因此需要为每个需要登录的服务器创建 publicKey。并在客户端的 ~/.ssh/config 文件中配置。
